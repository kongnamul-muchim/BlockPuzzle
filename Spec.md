# Spec — 10×10 블록 퍼즐 (가제: ChainCrush)

> 마지막 업데이트: 2026-05-13
> 프로젝트: 블록 연계 퍼즐 게임
> 아키텍처: 순수 C# Core + Unity Shell + DI 컨테이너
> 상태: 기획 단계

---

## 1. 게임 개요

| 항목 | 결정 |
|------|------|
| 장르 | 2D 블록 연계 퍼즐 |
| 엔진 | Unity (렌더링/입력/오디오 쉘) |
| 코어 로직 | 순수 C# (UnityEngine 비의존) |
| 승리 조건 | 없음 (생존 기반, 최고 점수 갱신) |
| 패배 조건 | 블럭이 1행 위로 넘어감 |
| 필드 크기 | 10×10 (가로 10열 × 세로 10행) |
| 블럭 생성 | 하단(10행)에서 생성, 위로 밀어올림 |
| 블럭 제거 | 인접한 같은 종류 블럭 2개 이상 클릭 |
| 낙하 | 제거된 블럭 위에 있던 블럭이 아래로 떨어짐 |
| 열 이동 | 빈 열 발생 시 중앙 방향으로 이동 |
| 턴 시스템 | 3회 제거 = 1행 추가 생성 |
| 난이도 | Easy(4색) / Normal(5색) / Hard(6색) |

### 핵심 메커니즘
- **전략적 제거**: 블럭을 무작정 지우면 위 블럭이 섞여서 연계 불가 → 신중한 선택 필요
- **연계 보너스**: 4개 이상 연계 시 기하급수적 점수 상승
- **낙차 보너스**: 블럭이 떨어진 높이만큼 추가 점수
- **난이도별 색상**: 4/5/6색 → 난이도와 전략성 조절

## 2. 게임 규칙 상세

### 2.1 필드 구조

```
     ← 게임오버 (1행 위로 넘어감)
┌──────────────────────────────┐
│ 1행 [ ][ ][ ][ ][ ]│[ ][ ][ ][ ][ ]│  ← 상단 (위험선)
│ 2행 [ ][ ][ ][ ][ ]│[ ][ ][ ][ ][ ]│
│ 3행 [ ][ ][ ][ ][ ]│[ ][ ][ ][ ][ ]│
│ 4행 [ ][ ][ ][ ][ ]│[ ][ ][ ][ ][ ]│
│ 5행 [ ][ ][ ][ ][ ]│[ ][ ][ ][ ][ ]│
│ 6행 [ ][ ][ ][ ][ ]│[ ][ ][ ][ ][ ]│
│ 7행 [ ][ ][ ][ ][ ]│[ ][ ][ ][ ][ ]│
│ 8행 [ ][ ][ ][ ][ ]│[ ][ ][ ][ ][ ]│
│ 9행 [ ][ ][ ][ ][ ]│[ ][ ][ ][ ][ ]│
│10행 [R][B][G][ ][ ]│[ ][ ][ ][ ][ ]│  ← 하단 (블럭 생성 지점)
└──────────────────────────────┘
     ↑ 블럭이 차오름
```

- **행(Row)**: 가로 줄, 1행(상단) ~ 10행(하단)
- **열(Column)**: 세로 줄, 좌 5열 + 우 5열 (짝수)
- 안전 구역: 1행 ~ 10행
- 1행 위로 블럭이 밀려나면 → **게임 오버**

### 2.2 블럭 생성

- 블럭은 항상 **10행(하단)** 에서 생성됨
- 3회 제거 시 1행 분량의 블럭이 하단에 추가 생성
- 생성되는 블럭의 색상은 해당 난이도의 색상 풀에서 랜덤 선택
- 새 블럭이 추가될 때 기존 블럭은 1행씩 위로 밀려남

### 2.3 블럭 제거

- **클릭**으로만 제거 가능
- 제거 조건: 인접한(상/하/좌/우) 같은 색상 블럭이 **2개 이상**
- 2개 미만인 블럭은 제거 불가
- 제거된 블럭의 위에 있던 블럭들은 **아래로 낙하**
- 연쇄 반응: 낙하 후 새로 인접한 같은 색 블럭이 생겨도 **자동 제거되지 않음** → 플레이어가 다시 클릭해야 함

### 2.4 열 이동

- 한 열(세로 줄)의 모든 블럭이 제거되면 → **중앙 방향으로 열 이동**
- 왼쪽 열이 비면 → 오른쪽 열들이 왼쪽으로 이동
- 오른쪽 열이 비면 → 왼쪽 열들이 오른쪽으로 이동
- 열 이동 시점: 블럭 제거 + 낙하가 완전히 끝난 후

```
예: 우측에서 2번째 열 제거
초기:          제거 후:
[ ][ ][ ][X][ ]    [ ][ ][ ][ ][ ]
[ ][ ][ ][X][ ] →  [ ][ ][ ][ ][ ]
[ ][R][G][X][B]    [ ][R][G][B][ ]
                    ↑ 빈 자리는 중앙 방향 이동
```

### 2.5 턴 시스템

| 조건 | 결과 |
|------|------|
| 블럭 3회 제거 | 1행 분량 새 블럭 하단 추가 |
| 제거 횟수는 모든 제거를 합산 | 연계 제거도 각각 1회로 카운트 |

### 2.6 게임 오버 조건

- 1행 위로 블럭이 밀려나면 즉시 게임 오버
- 게임 오버 시 현재 점수 DB 저장 (플레이어명 입력)

## 4. 점수 시스템

### 4.1 기본 점수

```
score = blockCount × baseScore × multiplier + fallBonus
```

| 변수 | 값 | 설명 |
|------|-----|------|
| blockCount | 제거한 블럭 개수 | |
| baseScore | 10 | 블럭 1개당 기본 점수 |
| multiplier | 연계 배율 (아래 표) | |
| fallBonus | 낙차 보너스 | |

### 4.2 연계 배율

| 연계 블럭 수 | 배율 | 실제 점수 (baseScore=10) |
|-------------|------|------------------------|
| 2개 | ×1.0 | 20점 |
| 3개 | ×1.0 | 30점 |
| **4개** | **×1.5** | **60점** ← 보너스 시작 |
| 5개 | ×2.0 | 100점 |
| **6개** | **×2.5** | **150점** |
| 7개 | ×3.0 | 210점 |
| 8개 | ×4.0 | 320점 |
| 9개 | ×4.5 | 405점 |
| **10개** | **×5.0** | **500점** |
| 11개+ | +0.5씩 증가 | 제한 없음 |

### 4.3 낙차 보너스

```
fallBonus = fallDistance × 5
```

| 낙하한 칸 수 | 보너스 |
|-------------|--------|
| 1칸 | +5점 |
| 3칸 | +15점 |
| 5칸 | +25점 |
| 8칸 | +40점 |
| 10칸 | +50점 |

### 4.4 누적 점수 표시

- 게임 중: 현재 점수, 현재 연계 수, 최근 낙차 표시
- 게임 오버: 최종 점수, 제거한 총 블럭 수, 최대 연계 표시

## 5. DB 연동 (랭킹 시스템)

### 5.1 아키텍처

```
Unity (WebGL) / Web
     ↓ HTTP API
Vercel Serverless (Next.js API)
     ↓ Prisma
Neon (PostgreSQL)
```

### 5.2 DB 모델

```prisma
model LeaderboardEntry {
  id         String   @id @default(cuid())
  playerName String
  score      Int
  maxCombo   Int      // 최대 연계 수
  totalCleared Int    // 제거한 총 블럭 수
  difficulty String   // "Easy" | "Normal" | "Hard"
  createdAt  DateTime @default(now())
  gameDuration Int    // 생존 시간 (초)

  @@index([score])       // 랭킹 정렬
  @@index([difficulty])  // 난이도별 필터
  @@index([createdAt])   // 최근 기록
}
```

### 5.3 API 엔드포인트

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | /api/leaderboard | 랭킹 조회 (상위 100, 난이도 필터) |
| POST | /api/leaderboard | 점수 저장 |

### 5.4 랭킹 표시

- 포트폴리오 사이트에 랭킹 페이지 추가
- 게임 내(Unity)에서도 랭킹 표시 (WebGL iframe 또는 자체 UI)

## 7. 아키텍처

### 7.1 레이어 구조

```
┌──────────────────────────────────────┐
│  Unity Layer (MonoBehaviour)         │
│  - GridRenderer                      │
│  - BlockRenderer                     │
│  - InputDetector                     │
│  - AudioController                   │
├──────────────────────────────────────┤
│  Core Interfaces                     │
│  IGridProvider, IInputProvider ..    │
├──────────────────────────────────────┤
│  Pure C# Game Logic                  │
│  Grid, Block, Chain, Score, GameState│
└──────────────────────────────────────┘
```

### 7.2 핵심 규칙
- `Assets/Core/` 내 모든 파일: `using UnityEngine` 절대 금지
- 모든 서비스는 인터페이스 뒤에 숨김 (DIP)
- MonoBehaviour는 순수 어댑터, 게임 로직 없음
- 의존성은 생성자 주입으로 해결
- Unity → Core 단방향 의존성

## 9. 구현 단계 (예상)

### Phase 1: Core 로직
- Grid.cs — 10×10 격자 관리, 블럭 생성/이동/제거
- Block.cs — 블럭 데이터 (색상, 위치)
- ChainDetector.cs — 인접 블럭 탐색 (BFS)
- ScoreManager.cs — 점수 계산
- GameStateMachine.cs — 게임 상태 전이
- DIContainer.cs + GameManager.cs

### Phase 2: Unity Adapter
- UnityGridRenderer — 격자 그리기
- UnityBlockRenderer — 블럭 스프라이트/애니메이션
- UnityInputDetector — 마우스 클릭 감지
- UnityAudioController — 효과음

### Phase 3: UI
- GameHUD — 점수, 연계, 남은 제거 횟수
- MainMenu — 난이도 선택
- GameOverScreen — 결과 + 이름 입력
- LeaderboardUI — 랭킹 표시

### Phase 4: DB 연동
- Vercel Next.js API 구축
- LeaderboardEntry 모델
- 게임 ↔ DB 점수 저장/조회
